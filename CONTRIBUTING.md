# 贡献指南

感谢关注 Spec2Case。这个项目优先关注真实需求到测试用例的稳定生成质量，欢迎提交问题、建议和改进。

## 反馈问题

提交 Issue 时建议包含：

- 使用场景：文本需求或图片需求。
- 输入类型：PRD、截图、原型图或补充备注。
- 期望结果和实际结果。
- 可复现步骤、日志或脱敏截图。

请不要提交 API Key、模型密钥、真实客户数据、未脱敏需求文档或生产数据库。

## 提交代码

提交 PR 前建议先确认：

- 不包含 `config/OAI_CONFIG_LIST`、`.env`、`data/`、`uploads/`、`outputs/`、`logs/` 等本地运行数据。
- 文档链接可访问。
- `docker compose config` 可以通过。
- 关键 Python 文件能正常编译。

## 设计原则

- 需求事实必须来自原始输入和人工确认。
- Prompt 和 Agent 定义尽量清晰、稳定、可追溯。
- 测试用例可以发散，但不能偏离需求事实。
- UI 改动应优先保证主要工作流清楚可用。
