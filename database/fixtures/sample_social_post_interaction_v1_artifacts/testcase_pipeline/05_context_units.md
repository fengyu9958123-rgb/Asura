# Context Units

PRD Knowledge 将 normal_lu 映射为普通用例生成单元，将 integration_lu 映射为跨模块链路用例生成单元；normal_lu 回填主需求 BLOCK，integration_lu 回填链路证据 BLOCK 与 flow_hint/flow_outline；support_only_blocks 默认不进入写用例上下文。

## LU-001 图文发帖输入、上传与发布结果
- unit_type: normal_lu
- evidence_blocks: B-001
- boundary: 覆盖发现-发帖入口下的文本和图片内容输入、图片立即上传、预览删除、发布必填校验，以及发布成功跳转和失败停留提示。

## LU-002 动态流列表、排序、分页、空态与卡片展示
- unit_type: normal_lu
- evidence_blocks: B-002
- boundary: 覆盖首页推荐/关注动态流的最新与热度排序、下拉加载更多、关注 tab 与通用空态引导，以及动态卡片基础信息、正文、九宫格图片和互动计数展示。

## LU-003 点赞与取消点赞交互
- unit_type: normal_lu
- evidence_blocks: B-003
- boundary: 覆盖动态卡片/帖子详情点赞按钮的登录校验、点赞/取消点赞即时状态与计数变化、异步请求期间防抖，以及落库失败后的状态回滚和提示。

## LU-004 评论与回复发送、删除和折叠展开
- unit_type: normal_lu
- evidence_blocks: B-004
- boundary: 覆盖帖子详情评论区中一级评论和二级回复的发送、长度限制、发送成功定位、评论排序、删除二次确认，以及一级评论删除后的子回复折叠和展开查看。

## LU-005 个人主页展示与关注操作
- unit_type: normal_lu
- evidence_blocks: B-005
- boundary: 覆盖从动态卡片头像/昵称进入个人主页后的用户信息展示、发帖列表分页排序与空态，以及关注/取消关注的登录校验、即时计数更新和失败不回滚规则。

## INT-001 发帖成功进入详情后进行评论互动
- unit_type: integration_lu
- flow_hint: 从发现页完成图文发帖开始，发布成功自动进入该帖子详情页，随后在评论区发送评论或回复，并验证新内容在详情页内按规则定位和展示的一条完整业务链路
- flow_outline: 在发帖页输入文本或选择图片并发布 -> 发布成功后自动跳转至该帖子详情页 -> 在详情页评论区发送一级评论或二级回复 -> 发送成功后页面定位至新内容，并保持评论列表排序规则
- excluded_atomic_focus: 不单独验证文本字数上限；不单独验证图片数量或大小边界；不单独验证评论输入长度上限；不单独验证删除评论的二次确认
- evidence_blocks: B-001, B-004
- boundary: 从发现页完成图文发帖开始，发布成功自动进入该帖子详情页，随后在评论区发送评论或回复，并验证新内容在详情页内按规则定位和展示的一条完整业务链路

## INT-002 动态流卡片点赞后的计数展示一致性
- unit_type: integration_lu
- flow_hint: 从首页动态流列表中的帖子卡片开始，用户对卡片执行点赞或取消点赞，最终验证卡片上的点赞状态与点赞数按即时响应或异常回滚规则保持一致的一条完整业务链路
- flow_outline: 进入首页推荐或关注动态流并查看帖子卡片互动计数 -> 对某条动态卡片点击点赞或取消点赞按钮 -> 异步请求返回前观察按钮防抖与本地计数状态 -> 根据请求成功或失败结果验证卡片点赞状态和点赞数一致性
- excluded_atomic_focus: 不单独验证动态流排序规则；不单独验证卡片头像昵称等静态字段展示；不单独验证未登录点赞引导
- evidence_blocks: B-002, B-003
- boundary: 从首页动态流列表中的帖子卡片开始，用户对卡片执行点赞或取消点赞，最终验证卡片上的点赞状态与点赞数按即时响应或异常回滚规则保持一致的一条完整业务链路

## INT-003 从动态流进入个人主页并执行关注操作
- unit_type: integration_lu
- flow_hint: 从首页动态流卡片的头像或昵称入口开始，进入对应用户个人主页，执行关注或取消关注，并验证主页关注状态及相关计数按规则更新的一条完整业务链路
- flow_outline: 在首页动态流查看包含头像和昵称的帖子卡片 -> 点击卡片头像或昵称进入该用户个人主页 -> 在个人主页查看用户信息和发帖列表 -> 执行关注或取消关注并验证本地状态及计数更新规则
- excluded_atomic_focus: 不单独验证动态卡片字段展示；不单独验证个人主页发帖列表分页；不单独验证个人主页无帖子空态；不单独验证未登录关注引导
- evidence_blocks: B-002, B-005
- boundary: 从首页动态流卡片的头像或昵称入口开始，进入对应用户个人主页，执行关注或取消关注，并验证主页关注状态及相关计数按规则更新的一条完整业务链路
